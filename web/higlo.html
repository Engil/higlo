Higlo comes with a command line tool, higlo. It reads source code files and output this code syntactically highlighted. If no source code file is given on the command line, then standard input is read instead. The source language (the lexer used) and the output format can be specified on the command line. The default language is "ocaml" and default output format is "xml".

Compiled OCaml code can be loaded to add languages or output formats.

Usage:

higlo [options] [source files]

The mk-higlo script eases the creation of a custom executable, i.e. a higlo tool where additional code is already linked in.

Suppose we defined an additional printer in myprint.ml. The printer only prints the code in raw text. We do not forget to register the printer using Higlo_printers.register_printer:

We can create a new higlo tool already containing this module:

mk-higlo -o myhiglo myprint.ml

Now we can use the "raw" printer with the produced executable:

./myhiglo -f raw mycode.ml
